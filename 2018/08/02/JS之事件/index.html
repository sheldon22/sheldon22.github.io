JS之事件

发表于 2018-08-02 | 更新于: 2018-08-03 | 分类于 JS | | 本文总阅读量 次
字数统计: 1,484 | 阅读时长 ≈ 6

JavaScript和HTML的交互是通过事件实现的。JavaScript采用异步事件驱动编程模型，当文档、浏览器、元素或与之相关对象发生特定事情时，浏览器会产生事件。事件就是用户或浏览器自身执行的某种动作。比如click、load、mouseover等，都是事件类型(俗称事件名称)，而响应某个事件的方法就叫做事件处理程序或者事件监听器。我们需要提前定义好某些事件发生了该怎么处理，这个过程叫做绑定事件处理程序。

事件流描述的是从页面中接收事件的顺序。目前主要有三种模型。

事件冒泡：事件开始时由最具体的元素接收，然后逐层向上冒泡，直至document对象，称为冒泡法。

事件捕获：单击首先发生在document上，然后依次向下传递，最终到达最具体的元素，称为捕捉法。

DOM事件流：DOM2级事件规定事件流包括三个阶段，事件捕获阶段，处于目标阶段，事件冒泡阶段，首先发生的是事件捕获，为截取事件提供机会，然后是实际目标接收事件，最后是冒泡阶段。事件传播的路径是先从document到相关标签，然后回到document。

DOM0级事件处理程序

DOM0级事件处理程序就是把一个函数赋值给一个事件的处理程序属性。每个元素都有自己的事件处理程序属性，这些属性名称通常为小写，如onclick等，将这些属性的值设置为一个函数，就可以指定事件处理程序。DOM0级事件处理程序的缺点是围绕着每个事件目标对于每种事件类型只能添加一个事件处理程序。如果添加多个，则以最后一个为准。

如果想删除事件处理程序，只需把元素的onclick属性赋为null即可。

DOM2级事件处理程序

DOM2级事件定义了两个方法用于处理指定和删除事件处理程序的操作：
addEventListener和removeEventListener。
所有的DOM节点都包含这两个方法，并且它们都接受三个参数：事件类型(eventType)，事件处理函数(handler)，布尔值(true表示在捕获阶段调用事件处理程序，false则表示在事件冒泡阶段处理)。在事件里使用this，引用的是被触发的元素。通过addEventListener()的第三个参数，我们可以决定代码是否采用捕捉法来处理事件。为了让我们的代码适用于更多的浏览器，最好还是将其设置为false，即只使用冒泡法来处理事件。使用DOM2级事件处理程序的好处是可以添加多个事件处理程序，并按照添加时的顺序触发。

防止默认行为

在浏览器模型中，有些事件自身就存在一些预定义行为。例如，单击链接会载入另一个页面。对此，我们可以为该链接设置监听器，并使用preventDefault()方法禁用其默认行为。

事件代理

事件代理就是利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件。

为什么我们需要使用事件代理呢？想象一下这样一个场景，一个列表(ul)里面有若干个列表项(li)。现在我们要为每一个列表项绑定相同的点击事件。该怎么做呢？

a. 给每个li手动绑定事件，如果li的数量少的话，这个方法还可以使用，但是如果li的数量十分的多，那么这个方法就显得十分的繁琐，代码量也会很大。

b. 对ul里的每一个子元素进行遍历，循环绑定事件。此方法只能给现有的li绑定事件，如果ul后续增加了子元素，新增的子元素将无法被绑定。

在这种情况下，我们就可以使用事件代理来达到目标，即在其父元素上绑定事件，这样就实现了动态绑定。即使父元素后续添加了新的元素，也可以实现绑定。
