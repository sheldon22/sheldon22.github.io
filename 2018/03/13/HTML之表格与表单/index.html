HTML之表格与表单

发表于 2018-03-13 | 更新于: 2018-03-13 | 分类于 HTML | | 本文总阅读量 次
字数统计: 1,225 | 阅读时长 ≈ 4

表格

table标签提供了一种在HTML中创建表格的方法。表格由行中的数据单元格组成，表格中的列数就是行中数据单元格的个数。一般来说，表格没有格式。
具体应用：用<table>来开始这个表格，<tr>构成表格中的一行，<th>元素分别是某一列的表头(表头是前后排列的，看上去它们好像构成了HTML中的一列，但实际上我们是定义了整个表头行，表头默认用粗体显示)，而具体的表格数据则用<td>来表示。

表单

网站使用form标签与用户进行交互。表单可以把浏览者输入的数据传送到服务器端，这样服务器端程序就可以处理表单传过来的数据。
所有的HTML表单都以一个<form>元素开始, 这个元素正式定义了一个表单。<form>标签是成对出现的，以<form>开始，以</form>结束。
form标签不仅包含构成表单的所有元素，还会告诉浏览器当你提交表单时要把表单数据发送到哪里(及浏览器要用什么方法发送数据)。它必须包含两个元素，action和method。
action属性定义了在提交表单时所收集的数据位置(URL)，method属性定义了发送数据的HTTP方法(“get”或”post”)。file协议不支持post。
name属性规定input元素的名称，只有设置了name属性的表单元素，才能在提交表单时传递它们的值。另外在单选按钮中，要把一组的name值设置成一致的。
所有表单控件(文本框、文本域、按钮、单选框、复选框等)都必须放在<form></form>标签之间，否则用户输入的信息就提交不到服务器上。

method属性：浏览器使用的主要方法由两种：POST和GET。
POST和GET完成的任务是一样的，都是将你的表单数据从浏览器发送到服务器，不过它们采用了两种不同的方式。POST会打包你的表单变量，在后台把它们发送到服务器；GET也会打包你的表单变量，但会把这些数据追加到URL的最后，然后再向服务器发送请求。也就是说，利用POST，所有表单数据会作为请求的一部分发送，对用户是不可见的。但使用GET，表单数据会追加到URL本身，所以用户能看到表单数据。

input元素
type="radio"，相同的name值分成一组，成为单选状态。
type="checkbox“，checked属性表示默认选中。
type="textarea"，<textarea>元素会创建一个多行文本区，可以在其中输入多行文本，如果输入的文本在文本区内放不下，右边还会出现一个滚动条。<textarea>元素不是一个空元素，所以它有开始和结束的标记。
type="hidden"，用户看不到显示效果，但是表单确实存在，依旧可以向服务器发送数据，可用于安全性校验。
type="submit"，把form所有数据打包发送给后端服务器。
type= "reset"，具有重置的作用。
type= "number"会创建一个只允许输入数字的单行文本输入控件。可以使用它可选的属性来指定这个元素允许的最大数和最小数。
type= "range"会显示一个滑动条而不是输入框。
type= "file"会创建一个文件输入控件，允许你上传文件。表单提交时，文件的内容会随其余的表单数据一同发送给服务器，使用这个元素的前提是必须使用Post方法。
type="placeholder"用来显示在输入框提示的信息。这个属性的值会比增加到控件的内容要浅一些, 一旦输入内容后, 原占位文本就会消失, 不会与输入的内容混杂在一起。
type="required"可以用于任何表单控件。有些浏览器在你提交表单之前会强制要求你在域中输入数据，不然表单将无法提交。
<select>元素会在web页面创建一个下拉菜单，这个下拉菜单包含一个或多个<option>元素。<option>元素定义了下拉菜单中的菜单项。设置selected="selected"属性即为默认选中。输入它时，前面需要空格。在<select>标签中设multiple="multiple"属性，就可以实现多选功能，在屏幕上显示所有的选项(如果选项太多，还会有一个滚动条)。windows操作系统下，按下Ctrl键同时进行单击可以进行多选。在Mac下使用Command +单击，可以选择多个选项。
