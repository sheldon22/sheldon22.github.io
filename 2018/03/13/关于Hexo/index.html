搭建hexo

搭建Hexo博客的基础流程为：安装Node.js →安装Git → 安装Hexo → 安装主题 → 本地测试运行 → 注册github并创建同名仓库 → 部署到github上开启page。

安装Hexo

npm install -g hexo-cli(如果出现报错，可能是权限问题，右键点击gitbash以管理员身份运行)。

初始化

在磁盘里随便建一个文件夹，名字随便取，然后进入刚建立的文件夹里，hexo init(hexo会在目标文件夹建立网站所需要的所有文件)。npm install(安装依赖包)。

在本地启动

hexo generate(生成静态文件)，hexo server(在本地服务器运行)

部署到githup上

在githup上创建一个新的仓库(Repository), 仓库名(Repository name)必须为你的用户名.github.io。

安装Git部署插件

npm install hexo-deployer-git –save

更改hexo配置文件

a. 把第 6 行的 title 改成你想要的名字
b. 把第 9 行的 author 改成你的大名
c. 把最后一行的 type 改成 git
d. 在最后一行，与 type 平齐，加上一行 repo: 仓库地址

打开GitHub Pages

打开 GitHub Pages 功能，如果已经打开了，就直接点击预览链接。

更换主题

在https://github.com/hexojs/hexo/wiki/Themes
找到你喜欢的主题，clone它的SSH。

安装主题

Hexo配置文件

在Hexo中有两份配置文件，其名称都是_config.yml。其中，一份位于hexo站点的根目录下，主要包含Hexo本身的配置；另一份位于主题目录下，这份配置由主题作者提供，主要用于配置主题相关的选项。

启用主题

打开网站配置文件_config.yml，找到theme(第75行)，将其值更改为你使用的主题名。 hexo generate,hexo deploy后等一分钟，然后刷新你的博客页面，你会看到一个新的外观。

在博客上插入图片

a. 把主页配置文件_config.yml里的post_asset_folder设置为true。
b. 在hexo目录下执行npm install hexo-asset-image --save下载安装一个可以上传本地图片的插件。
c. 等待一小段时间后，再运行hexo new "xxxx"来生成md博文时，/source/_posts文件夹内除了xxxx.md文件还有一个同名的文件夹。
d. 之后想在xxxx.md中引入图片时，先把图片复制到xxxx这个文件夹中，![你想输入的替代文字](xxxx/图片名.jpg)。xxxx是这个md文件的名字，也是同名文件夹的名字。

绑定私人域名

购买域名

在namesilo上搜索自己想要的域名(namesilo需要注册帐号，如果没有帐号，先申请一个，资料都可以随便填，但邮箱最好保证正确)。网址https://www.namesilo.com/

找到合适的域名后选中它，点击register…

第三项是是否自动扣款，第四项是隐私设置，第五项是设置年限。

全部改好以后就可以付款了，选支付宝。

付完款以后登录你刚才申请的帐号，进入

选择
点开后点击它解析DNS，挂到github上就好。

解析域名

记录类型选A或CNAME

A的记录值就是ip地址。github提供了两个IP地址，192.30.252.153和192.30.252.154，这两个IP地址为github的服务器地址，两个都要填上。hostname空着就行。

CNAME记录值填你的账户名github.io

添加CNAME文件

在你的hexo文件夹下source文件夹中新建一个CNAME文件(必须大写，无后缀名)，然后用文本编辑器打开，在首行添加你的网站域名如xxxx.com，注意前面没有http://，也没有www，然后使用hexo generate && hexo deploy上传部署。

了解Hexo

a. _config.yml是整体的配置文件，很多基础配置、插件配置等都需要在里面进行。要注意的是，该文件格式要求极为严格，缺少一个空格都会导致运行错误。小提示：不要用 Tab 缩进，用两个空格符。

b. layout，包括draft(草稿)、page(页面)、post(文章)。其中post是你发表的文章，会显示在你的网站里，一篇post包括title 标题、date 日期、tags 标签等信息。draft是草稿，只有在本地才能看到，publish后才能成为post，draft和post差别是date，因为draft没有发表，所以不需要指定日期。page是一个页面，对应一个新的html页面。

c. public文件夹，这个文件夹是最终会发布到网站上的内容。用户只能看到public文件夹里面的东西，而其他的source、themes等都是为public服务的。hexo generate把代码里的所有配置信息、source里写的文章、themes里的样式都结合起来，生成最终的静态html文件，存入public文件夹内。在我们执行了发布指令hexo deploy后，就会把public的内容部署到GitHub Pages上。当用户在访问我们的博客时，他们会看到public里生成的html文件。即代码和真实静态页面是独立的。
