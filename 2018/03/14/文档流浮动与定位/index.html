文档流、浮动与定位

发表于 2018-03-14 | 更新于: 2018-03-17 | 分类于 CSS | | 本文总阅读量 次
字数统计: 1,184 | 阅读时长 ≈ 4

文档流

文档流指的是浏览器在页面放置HTML元素所用的的方法。

块元素从上向下流，各个块元素之间会有一个默认的换行，每个块元素会占据页面的整个宽度。块级元素的宽度默认是自适应它的父元素的。块级元素的高度是由它内部文档流中元素的高度总和决定的。如果块级元素里面只有一个内联元素，那个块级元素的高度就是由这个内联元素的行高决定的。如果有多个内联元素，就是它们的行高相加。

内联元素在块元素内部从左上方流向右下方。如果流动遇到阻碍, 它就会自动换行继续流动，Span的border(汉字)如果在自动换行的过程中被截断了, 它并不会变成两个, 会变成上下各有一部分。而英文则不会换行, 如果想要它换行, 要添加一个Word-break: break-all。
内联元素的宽度是由文本里面字的个数确定的。高度是由行高决定的。

使用浮动float、绝对定位position: absolute和固定定位position: fixed可以让元素脱离文档流。

浮动

float被设计出来的初衷是用于文字环绕效果, 即对一个图片使用float之后，它周围的文字会环绕着它。后来发现使用float+div是一种不错的排版方法。所以现在绝大多数情况下使用float就是为了横向排版或者多列布局了。

由于float会使元素脱离文档流，所以当一个元素里只包含一个浮动元素时，这个浮动元素会破坏父元素的原本结构，使得父元素的高度发生坍塌。块级元素设置了float之后，如果不手动设置宽度，其宽度会根据内容自动调整。使用了float的元素会自动变为块元素。

使用浮动会带来一定的副作用，如父元素高度坍塌等，所以我们需要清除浮动。清除浮动的步骤是给包含浮动元素的容器设置clearfix

.clearfix:after: 使用after伪类给所有包含.clearfix类的元素里面创建一个虚拟子元素，该子元素为父元素的最后一个子元素。
content: ‘’是指这个伪元素的内容为空，content属性需要空引用作为它的值。
clear：both清除伪元素两边的浮动元素。
display: block 默认的伪元素是行内元素，但是下面的clear属性只对块元素起作用，所以用display将这个伪元素改为块元素。
*zoom: 1 ie8以下是不支持伪元素：after的，ie通常是触发hasLayout来清除浮动了，常见的就是zoom:1，*号是是IE8的一个bug，IE8看到以*开头的代码，会忽略星号，执行后面的代码。

定位

定位就是通过设置position属性值来覆盖默认的布局方式。position属性值如下

static

static是position属性的默认值，即没有定位。文档流里的元素都处在正常的位置。

relative

使用相对定位relative时，会导致元素相对自身位置发生变化，而覆盖掉其他的元素框。但是在文档流中，它仍然占据着原来的位置，所以其他元素的位置不会发生变化。

absolute

使用绝对定位absolute的元素，在没有父元素设置定位的话，默认就是以body来定位的。要想相对当前元素的父元素来定位，父元素需要设置position：relative。使用绝对定位absolute的元素具有收缩性，如果没有设置宽度，元素会根据内容自动调整宽度(absolute具有自适应宽度的特性)。元素的位置通过”left”, “top”, “right”以及”bottom”属性进行规定。绝对定位使元素脱离文档流，因此不占据空间。

fixed

fixed的特性跟absolute一样，唯一的区别在于fixed是根据浏览器窗口来定位，无论其他元素怎么设置，都影响不了它。

sticky

sticky是相对定位和固定定位的结合。默认情况下表现为相对定位，当浏览器窗口顶端与元素的距离等于top属性的值时，转变为固定定位。这种定位的兼容性不是太好，使用前可以先去can i use里查询下。
